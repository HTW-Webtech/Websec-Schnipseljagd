Firewall 2

  1. Auf Ubuntu ist standardmäßig eine einfache Firewall installiert, die jedoch deaktiviert ist. Die Firewall heisst ufw (user firewall). Aktivieren Sie die Firewall mit: sudo ufw enable

  2. Kontrollieren Sie die aktuellen Regeln mit:

    sudo ufw status verbose

    1. Machen Sie einen Screenshot.

    2. Beschreiben Sie die unter Default (deutsch: Vorgabe) angegeben Regeln mit eigenen Worten.


  1. Zum Definieren von Firewall-Regeln kann ein einfacher Befehlssyntax genutzt werden2:

    (sudo) ufw ZUGRIFFSRECHT RICHTUNG PROTOKOLL

    Dabei kann ZUGRIFFSRECHT auf allow, deny oder reject gesetzt werden, RICHTUNG auf in oder out und PROTOKOLL auf Protokolle der Anwendungsschicht3 (z.B. http, https, ftp, ssh, smtp, imap,...).

    1. Sperren Sie alle ausgehenden http-Verbindungen. Welchen Befehl nutzen Sie?

    2. Versuchen Sie, folgende Internetseiten mit dem Firefox zu öffnen:

    • http://fiw.htw-berlin.de/

    • https://lsf.htw-berlin.de

      Machen Sie jeweils einen Screenshot.

  1. Löschen Sie den Firewalleintrag wieder. Dazu nehmen Sie Ihren Befehl, mit dem Sie die Regel definiert haben und fügen direkt hinter ufw das Wort delete ein. Machen Sie einen Screenshot von Befehl und Ausgabe.

Webserver


  1. Als Webserver ist der apache2 installiert. Überprüfen Sie, ob der Webserver bereits läuft (z.B. mit ps).

    1. Machen Sie einen Screenshot, aus dem ersichtlich wird, ob er bereits läuft oder nicht.

    2. Falls der Webserver noch nicht läuft, starten Sie ihn mit

      sudo service apache2 restart und machen Sie einen Screenshot mit Befehl und Ausgabe.

  2. Starten Sie den Firefox und lassen Sie sich Ihren Webserver mit http://localhost anzeigen. Machen Sie einen Screenshot.

  3. Die angezeigte Datei index.html liegt im Wurzelverzeichnis des Webservers: /var/www/.

    1. Geben Sie die Zugriffsrechte für das Verzeichnis und die Datei index.html an.

    2. Beschreiben Sie den Weg, wie Sie die Zugriffsrechte ermittelt haben.


Absolute Directory Traversal Attack

  1. Laden Sie sich das Archiv DSDS-UE-Webattacks.tgz herunter (in die virtuelle Maschine!). Kopieren Sie es nach /var/www/ und entpacken Sie es dort mit dem Befehl tar xvfz DATEINAME. Sie benötigen jeweils sudo-Rechte!

  2. Schauen Sie sich die Dateien webform-traversal1.php und response-traversal1.php an (z.B. mit dem gedit) und versuchen Sie sie zu verstehen.

  3. Öffnen Sie das Script webform-traversal1.php unter http://localhost/webform-traversal1.php im Browser.

  4. Geben Sie im Namensfeld einen beliebigen Text ein, und im Dateifeld welcome.txt und schicken Sie die Eingabe ab. Machen Sie einen Screenshot von der Antwortseite.

  5. Gehen Sie zurück zur Eingabemaske und geben Sie im Dateifeld /var/www/welcome.txt ein. Welches Ergebnis erhalten Sie?

  6. Versuchen Sie nun, auf die passwd-Datei zuzugreifen, eine Systemdatei unter Linux, in der alle User angegeben sind.

    1. Wo liegt sie? Geben Sie den Weg an, wie Sie sie gefunden haben und geben Sie den Pfad an.

    2. Machen Sie jeweils einen Screenshot von ausgefüllter Eingabemaske und Ergebnisseite.

  7. Versuchen Sie nun, auf die shadow-Datei zuzugreifen, in der die Passwörter verschlüsselt abgelegt sind. Sie liegt im gleichen Verzeichnis wie passwd.

    1. Machen Sie jeweils einen Screenshot von ausgefüllter Eingabemaske und Ergebnisseite.

    2. Was für ein Unterschied zwischen den beiden Dateien ist für das unterschiedliche Verhalten verantwortlich? Tipp: Kontrollieren Sie die Zugriffsrechte.
